Slide 2–4: Data Understanding

Insurance Dataset:

* The target column was claim_status (0 = not approved, 1 = approved)
* Challenge: It was very imbalanced – only 1.4% of people had approved claims
* Some columns like agent_id had thousands of unique values – we call this high cardinality

Apartment Dataset:

* Contained 156,000 training rows and 39,000 test rows
* The goal was to predict price_z – the apartment price
* There were missing values in some columns like build_mat, cond_class, and has_lift
* Features included size, number of rooms, location, year built, nearby services, etc.

Slide 5–6: Preprocessing

Let me explain how I prepared the data.

For Insurance:
* I used Target Encoding for variables with many unique values like agent_id and entity_id. This helps convert categories into useful numbers bas
ed on the average of the target.
* For simple categorical columns like gender or channel, I used One-Hot Encoding
* The big issue was class imbalance. I fixed it using SMOTE, a technique that creates synthetic examples to make the dataset 50% approved and 50% not approved.
* After all this, we had a clean and balanced dataset.

For Apartments:

* For missing values: I used mean for numbers and most frequent for categories.
* I used OneHotEncoder for variables like obj_type and own_type
* Applied StandardScaler to normalize numerical columns.
* I also removed unnecessary columns like unit_id before modeling.
* I checked the correlation of features with price_z. Variables like market_volatility were highly correlated with price, while others like crime_rate were negatively related.

Insurance Project (Classification):

* I tried several models: Logistic Regression, KNN, XGBoost, and Random Forest.

Apartment Project (Regression):
